42 /** @brief Class for data to be shared in a thread-safe manner between tasks.
43  * @details This class implements an item of data which can be shared between
44  * tasks without the risk of data corruption associated with global
45  * variables. Unlike queues, shares do not use a buffer for many data
46  * items; there is only a one-item buffer in which the most recent
47  * value of the data is kept. Shares therefore do not provide the
48  * task synchronization or incur the overhead associated with queues.
49  *
50  * The data is protected by using critical code sections (see the
51  * FreeRTOS documentation of @c portENTER_CRITICAL() ) so that tasks
52  * can't interrupt each other when reading or writing the data is
53  * taking place. This prevents data corruption due to thread
54  * switching. The C++ template mechanism is used to ensure that only
55  * data of the correct type is put into or taken from a shared data
56  * item. A @c TaskShare<DataType> object keeps its own separate copy
57  * of the data. This uses some memory, but it is necessary to
58  * reliably prevent data corruption; it prevents possible side
59  * effects from causing the sender's copy of the data from being
60  * inadvertently changed.
61  *
62  * @section usage_share Usage
63  * The following bits of code show how to set up and use a share to
64  * transfer data of type @c uint16_t from one hypothetical task
65  * called @c task_A to another called @c task_B.
66  *
67  * In the file which contains @c setup() we create a shared data
68  * object. The constructor of the @c Share<uint16_t> class is given
69  * a name for the share; the name will be shown on system diagnostic
70  * printouts. If it is desired to print diagnostic information to a
71  * serial monitor, a pointer to an output stream may also be given:
72  * @code{.cpp}
73  * #include "taskshare.h"
74  * ...
75  * /// Data from sensor number 3 on the moose's right antler
76  * Share<uint16_t> my_share ("Data_3", &Serial);
77  * @endcode
78  * If there are any tasks which use this share in other source files,
79  * we must re-declare this share with the keyword @c extern near the
80  * top of those files to make it accessible to those task(s). This
81  * copy of the share does not need a Doxygen comment:
82  * @code
83  * // Sensor 3 (right antler) data
84  * extern Share<uint16_t> my_share;
85  * @endcode
86  * In the sending task, data is put into the share:
87  * @code
88  * uint16_t a_data_item = 42; ///< Holds antler data
89  * ...
90  * a_data_item = antler3 (); // Get the data
91  * my_share.put (a_data_item); // Put data into share
92  * @endcode
93  * In the receiving task, data is read from the share:
94  * @code
95  * uint16_t got_data; ///< Holds received data
96  * ...
97  * my_share.get (got_data); // Get local copy of shared data
98  * @endcode
99  */

195 /** @brief Put data into the shared data item.
196  * @details This method is used to write data into the shared data item. It's
197  * declared @c inline so that instead of a regular function call at
198  * the assembly language level, <tt>an_object.put (x);</tt> will
199  * result in the code within this function being inserted directly
